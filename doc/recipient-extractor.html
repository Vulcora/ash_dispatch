    Recipient Field Extraction

    View Source

    The AshDispatch.Event.RecipientExtractor provides a flexible, configuration-driven approach to extracting recipient identifiers and display names from any data structure, eliminating hardcoded field assumptions.

    API Documentation: For complete function documentation, see AshDispatch.Event.RecipientExtractor.

    Overview

    The RecipientExtractor eliminates the need for hardcoded field names like :email or :contact_person. Instead, you configure which fields to use once, and the system works with any recipient data structure - User structs, external contacts, leads, or even plain maps.

    Quick Start

    1. Basic Configuration (Zero Config)

    For most applications, you only need to configure generic defaults:

    # config/config.exs
config :ash_dispatch,
  recipient_fields: [
    identifier: :email,
    name: :contact_person
  ]

    This works for all transports (email, SMS, in-app, etc.) by default.

    2. Transport-Specific Overrides

    If different transports need different fields:

    config :ash_dispatch,
  recipient_fields: [
    # Generic defaults
    identifier: :email,
    name: :contact_person,

    # SMS uses phone number instead of email
    sms: [
      identifier: :mobile_phone,
      name: :first_name
    ]
  ]

    3. Audience-Specific Overrides

    If different user types have different field names:

    config :ash_dispatch,
  recipient_fields: [
    # Generic defaults
    identifier: :email,
    name: :contact_person,

    # Admins use different name field
    audiences: [
      admin: [
        name: :full_name
      ],

      # Leads use different email field
      lead: [
        identifier: :primary_email,
        name: :company_name
      ]
    ]
  ]

    4. Event-Level Overrides

    Override for specific events using DSL:

    # In your event module
defmodule MyApp.Events.SpecialEvent do
  use AshDispatch.Event

  dispatch do
    id "special.event"
    domain :custom

    # Override recipient fields for this event only
    recipient do
      email do
        identifier :alternate_email
        name :display_name
      end
    end

    channels do
      channel :email, :user
    end
  end
end

    Or in resource-based events:

    # In your resource
dispatch do
  event :special_case,
    trigger_on: :special_action,
    recipient: [
      email: [
        identifier: :backup_email,
        name: :preferred_name
      ]
    ]
end

    Advanced Field Formats

    Nested Fields

    config :ash_dispatch,
  recipient_fields: [
    identifier: {:field, [:contact, :email]},
    name: {:field, [:contact, :full_name]}
  ]

    Works with structs like:

    %User{
  contact: %{
    email: "user@example.com",
    full_name: "John Doe"
  }
}

    String Keys (for JSON data)

    config :ash_dispatch,
  recipient_fields: [
    identifier: {:string_field, "email_address"},
    name: {:string_field, "display_name"}
  ]

    Custom Extraction Functions

    config :ash_dispatch,
  recipient_fields: [
    identifier: &MyApp.extract_primary_contact/1,
    name: &MyApp.format_recipient_name/1
  ]
    defmodule MyApp do
  def extract_primary_contact(recipient) do
    recipient.emails
    |> Enum.find(&(&1.primary == true))
    |> Map.get(:address)
  end

  def format_recipient_name(recipient) do
    "#{recipient.first_name} #{recipient.last_name}"
  end
end

    Resolution Order (Cascading)

    The system checks configurations in this order (most specific to least specific):

    1. Event DSL override - Configured in the event definition
    2. Audience + Transport - audiences: [admin: [email: [identifier: :admin_email]]]
    3. Transport - sms: [identifier: :mobile_phone]
    4. Generic default - identifier: :email
    5. Error - Clear error message if no config found

    Error Messages

    If extraction fails, you'll get a clear error showing the resolution path:

    No identifier field configured for email transport (user audience).

AshDispatch doesn't assume field names - you must configure them.

Add generic default to config/config.exs:

    config :ash_dispatch,
      recipient_fields: [
        identifier: :your_field_name
      ]

    Or if a field is missing from the recipient:

    Could not extract identifier from recipient for email transport (user audience).

Expected field: :email
Recipient type: MyApp.Accounts.Lead
Available keys: [:id, :primary_email, :company_name]

Configuration resolution:
1. Event DSL: not set
2. Audience (lead) + Transport (email): not set
3. Transport (email): not set
4. Generic default: :email
→ Resolved to: :email

Fix options:
1. Add field :email to your recipient struct
2. Change config to use an existing field (:primary_email)
3. Use custom extraction function

    Examples by Use Case

    E-commerce Platform

    config :ash_dispatch,
  recipient_fields: [
    identifier: :email,
    name: :full_name,

    # SMS for order updates
    sms: [
      identifier: :phone_number
    ]
  ]

    B2B Platform with Leads

    config :ash_dispatch,
  recipient_fields: [
    identifier: :email,
    name: :contact_person,

    audiences: [
      # Leads might use different email field
      lead: [
        identifier: :primary_email,
        name: :company_name
      ],

      # Partners have dedicated contact info
      partner: [
        identifier: {:field, [:primary_contact, :email]},
        name: {:field, [:primary_contact, :name]}
      ]
    ]
  ]

    CRM with External Contacts

    # For external contacts (not User structs)
config :ash_dispatch,
  recipient_fields: [
    # Custom function to extract from any contact type
    identifier: &MyApp.Contacts.get_email/1,
    name: &MyApp.Contacts.get_display_name/1
  ]

defmodule MyApp.Contacts do
  def get_email(%{email: email}), do: email
  def get_email(%{"email" => email}), do: email
  def get_email(contact) when is_binary(contact), do: contact

  def get_display_name(%{name: name}), do: name
  def get_display_name(%{"name" => name}), do: name
  def get_display_name(_), do: nil
end

    Testing

    The RecipientExtractor is fully testable:

    test "extracts identifier from custom struct" do
  recipient = %{
    id: "123",
    primary_email: "test@example.com",
    full_name: "Test User"
  }

  # With custom config
  config = %{
    recipient: %{
      email: %{
        identifier: :primary_email,
        name: :full_name
      }
    }
  }

  identifier = RecipientExtractor.extract_identifier(
    recipient,
    :email,
    :user,
    config[:recipient]
  )

  assert identifier == "test@example.com"
end

    Migration from Hardcoded Fields

    If you're migrating from a system with hardcoded fields, the transition is simple:

    Before:

    # Hardcoded in dispatcher
recipient_email = recipient.email

    After:

    # config/config.exs
config :ash_dispatch,
  recipient_fields: [
    identifier: :email,  # Same field, now configurable!
    name: :contact_person
  ]

    The system now works with any field name you configure.
