    Helper module for broadcasting counter updates.

    This module provides a unified interface for counter broadcasting that:

    • Uses configured counter registry and broadcaster implementations
    • Automatically detects if counters should broadcast to admins
    • Handles both user-specific and admin-wide broadcasts
    • Prevents duplicate broadcasts when the user is also an admin

    Configuration

    config :ash_dispatch,
  counter_registry: MyApp.CounterRegistry,
  counter_broadcaster: MyApp.CounterBroadcaster

    Usage

    # Simple user counter
CounterHelper.broadcast_counter(user_id, :cart_items)

# Counter that broadcasts to user AND admins
CounterHelper.broadcast_counter(user_id, :pending_orders)

# Admin-only counter
CounterHelper.broadcast_counter_to_admins(:pending_reseller_requests)

    broadcast_counter(user_id, counter_name)

    Smart counter broadcaster that automatically determines broadcast targets.

    Reads counter definition from configured registry to determine:

    1. Whether to broadcast to admins (based on broadcast_type)
    2. Query invalidation keys (from query_keys)

    For counters with broadcast_type: :user_and_admin:

    • Broadcasts personal count to user
    • Broadcasts total count to all admins

    For counters with broadcast_type: :user_only:

    • Broadcasts only to the specific user

    Parameters

    • user_id - User UUID to broadcast to
    • counter_name - Counter identifier (e.g., :cart_items, :pending_orders)

    Returns

    :ok or {:error, reason}

    broadcast_counter_to_admins(counter_name, opts \\ [])

    Broadcast a counter to all admins with query invalidation metadata.

    Options

    • :exclude_user_id - Skip broadcasting to this user (prevents duplicate when user is admin)

    Parameters

    • counter_name - Counter identifier (typically prefixed with admin_)

    Returns

    :ok or {:error, reason}
