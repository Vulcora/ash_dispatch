    AshDispatch.TemplateCompiler (AshDispatch v0.1.0)

    View Source

    DEPRECATED: This module is no longer needed. Templates are now automatically compiled by Mix.Tasks.Compile.AshDispatch.

    Migration

    Remove all manual template compilation calls:

    # OLD (no longer needed):
defmodule MyApp.Events.Orders.Created do
  use AshDispatch.Event
  require AshDispatch.TemplateCompiler
  AshDispatch.TemplateCompiler.compile_templates()  # ← Remove this!

  # ... rest of event module ...
end

# NEW (automatic):
defmodule MyApp.Events.Orders.Created do
  use AshDispatch.Event

  # Templates are auto-compiled during mix compile!
  # No manual calls needed.
end

    How It Works Now

    The Mix.Tasks.Compile.AshDispatch compiler automatically:

    1. Scans for templates in both module-based and convention-based locations
    2. Compiles all templates into AshDispatch.CompiledTemplates during mix compile
    3. Bundles them in production releases
    4. Falls back to file-based loading in development

    Benefits:

    • ✅ No manual macro calls needed
    • ✅ Works for pure inline DSL events (no event module required)
    • ✅ Consistent compilation for all templates
    • ✅ Simpler, cleaner event modules

    Legacy Support

    This module is kept for backwards compatibility but will be removed in a future release. The macro still works but is redundant - templates are compiled automatically regardless.

    Example Directory Structure

    lib/my_app/events/orders/created/
├── event.ex                    # Event module
└── templates/
    ├── email.html.heex         # Default HTML email
    ├── email.text.eex          # Default text email
    ├── email.admin.html.heex   # Admin-specific HTML
    └── email.admin.text.eex    # Admin-specific text

    Generated Function

    Creates a __compiled_templates__/0 function that returns:

    %{
  "email.html.heex" => "<html>...</html>",
  "email.text.eex" => "Plain text...",
  "email.admin.html.heex" => "<html>Admin version...</html>",
  "email.admin.text.eex" => "Admin text..."
}

    get_template(module, filename)

    Gets a specific compiled template by filename.

    Returns {:ok, content} or :error.

    list_templates(module)

    Lists all compiled template filenames.

    Returns empty list if templates haven't been compiled.
